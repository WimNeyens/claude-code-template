# Decisions log

Lightweight sibling to `docs/adr/`. Use this for decisions too small to warrant a full Architecture Decision Record but worth remembering: library choices, naming conventions, small trade-offs.

If a decision turns out to be load-bearing or contested, promote it to a full ADR in `docs/adr/` and link back from here.

Format:

```
## YYYY-MM-DD — <short title>
**Decision:** what was decided
**Why:** the reason, in one or two sentences
**Alternatives considered:** what else was on the table (optional)
```
