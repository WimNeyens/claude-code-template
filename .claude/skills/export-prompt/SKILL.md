# Export Prompt — Save to Library

Export a reusable prompt, plan, or skill template to `C:\DEV\Claude\library\` for cross-project reuse.

## Modes

### Interactive (default — `/export-prompt`)

1. **Ask what to export.** If the argument is omitted, ask the user: "What would you like to export? A harvest paste-prompt from this session, the current plan, or a skill template?"

2. **Determine the category** based on the answer:
   - Harvest paste-prompt → `prompts/harvest/`
   - Implementation plan → `plans/`
   - Skill template → `prompts/skills/`

3. **Continue to the export steps below.**

### Harvest mode (`/export-prompt harvest`)

1. **Locate the paste-prompts.** Look back through the current session for paste-prompts generated by `/harvest` (identified by the 🟢 copy markers or the paste-prompt structure: context paragraph, target file, placement, verbatim content).

2. **For each paste-prompt found**, continue to the export steps below with category `prompts/harvest/`.

### Plan mode (`/export-prompt plan`)

1. **Locate the current plan.** Check `C:\DEV\Claude\plans\` for the most recent plan file from this session.

2. **Continue to the export steps below** with category `prompts/plans/`.

## Export steps (all modes)

1. **Generate a filename.** Use the `<topic>_<descriptive-name>.md` convention:
   - Topic: short lowercase tag for the subject area (`git`, `auth`, `ci`, `hooks`, `docs`, etc.)
   - Name: kebab-case description of what the prompt does
   - Example: `git_basics-primer.md`, `auth_session-token-rewrite.md`

2. **Scan for project-specific terms.** Check the content for hardcoded project names, internal hostnames, specific file paths, or credentials. Flag any matches: "⚠ Contains project-specific term: '<term>' — generalize before exporting?" Let the user decide whether to proceed or edit first.

3. **Write the file** to `C:\DEV\Claude\library\<category>\<filename>`:
   - Start with a single-line HTML comment: `<!-- Purpose: <one-line description> -->`
   - Follow with the prompt content
   - Create parent folders if they don't exist

4. **Confirm:** "Exported to `library/prompts/<category>/<filename>`"

## What this skill does NOT do

- No auto-export — always requires explicit invocation.
- No scrubbing — flags project-specific terms but never rewrites content.
- No sync or indexing — the library is a flat collection of files.
- No session-start cost — loads only when explicitly invoked.
